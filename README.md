# STUDIO 50 — Design System

Цифровая дизайн-система сети бутик-фитнеса **STUDIO 50** (50-минутные групповые тренировки).
Адаптивная, доступная (WCAG 2.2 AA), с токен-архитектурой *primitive → semantic → component*.

**🔗 Живая версия:** `https://<ваш-логин>.github.io/<имя-репозитория>/`

---

## Что внутри

| Файл | Назначение |
|------|------------|
| `index.html` | Полная дизайн-система — один самодостаточный файл, работает офлайн и без сборки |
| `tokens.css` | Дизайн-токены (цвета, типографика, отступы, радиусы, тени, z-index) — можно импортировать в свои проекты |
| `README.md` | Этот файл |

Разделы системы: цвета · типографика · маркер-стиль · основа/отступы · сетка ·
стекло (glassmorphism) · компоненты · иконки · применение · доступность.

## Как опубликовать на GitHub Pages

1. Создайте новый репозиторий на GitHub (например `studio50-design-system`).
2. Загрузите в него **все файлы из этой папки** (`index.html`, `tokens.css`, `README.md`, `.nojekyll`).
3. Откройте **Settings → Pages**.
4. В разделе *Build and deployment* выберите:
   - **Source:** `Deploy from a branch`
   - **Branch:** `main` · папка `/ (root)` → **Save**
5. Через 1–2 минуты сайт будет доступен по адресу
   `https://<ваш-логин>.github.io/<имя-репозитория>/`

> Файл `.nojekyll` уже добавлен — он отключает обработку Jekyll, чтобы все файлы отдавались как есть.
